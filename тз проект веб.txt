Тех. задание
Проект: Менеджер заметок(веб-приложение) с rest api
Стек: Python, flask, SQLite, bootstrap
Цель проекта: Разработать веб приложение для создания и управления заметками с поддержкой файлов, фильтров, экспорта и REST api
Функционал:
1. Пользователи
Регистрация/вход/выход
Безопасность паролей(хеширование)
Приватные заметки
2. Заметки
Создание(Заголовок + текст + категория + избранное + файл)
Просмотр(Список и детальная страница)
Редактирование
Удаление(с подтверждением)
3. Категории - Работа, учеба, личное, заметки
4. Фильтрация
Поиск по заголовку и тексту
Фильтр по категориям и избранному
5. Файлы
Поддерживаемые форматы: txt, pdf, docx, jpg, jpeg, png, webp
Ограничение по размеру 10 мб
Возможность заменить файл
Автоматическое удаление файла при замене или удалении заметки
6. Экспорт
В txt и csv
7. REST Api(PyJWT)
Регистрация/логин(токен)
CRUD для заметок
Фильтрация и поиск
Список категорий и статистика
ПЛАН!!!
1. Создать новое вирт. окр. и установить зависимости(Flask, sqlalchemy, flask-login, pyjwt)
2. Сделать структуру проекта(папки routes, templates, static, uploads; файлы app.py, config.py, models.py, utils.py)
3. Настроить БД и конфигурация(класс SQLite, модели User, Note, Category, Attachment)
4. Сделать аутентификацию(маршруты /register, /login, /logout, шаблоны и хеширование паролей)
5. Сделать CRUD заметок(маршруты /notes/ /create /id, т.д)
6. Фильтрация(поиск, категории и избранное)
7. Добавить загрузку файлов(проверка расширений, сохранение, отображение и привязка к заметке)
8. Добавить экспорт в txt и csv
9. Реализовать REST Api(routes/api.py, jwt энд поинты, /api/notes, /api/categories, т.д)
10. Реализовать интерфейс(bootstrap 5, карточки, подтверждение удаления)
11. Тест(тестируем регистрацию, crud, фильтры, файлы, экспорт, api)
РЕЗУЛЬТАТ
Полностью рабочее веб приложение с работающим rest api
